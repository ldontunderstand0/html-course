## Правила каскадирования и аппаратно-зависимые таблицы стилей

## Что такое каскадирование

Каскадирование - это механизм CSS, который определяет, какие стили
будут применены к элементу, если есть несколько правил.

## Основные принципы каскада

### 1. Приоритет источника

Приоритет (сильнее → слабее):

1.  Inline-стили
2.  Внутренние стили (`<style>`)
3.  Внешние стили (`.css`)

``` html
<p style="color: red;">Текст</p>
```

### 2. Специфичность селекторов

Приоритет селекторов:

- `#id` → самый сильный
- `.class` → средний
- `tag` → слабый

``` css
p { color: blue; }
.text { color: green; }
#main { color: red; }
```

* будет красный

### 3. Порядок объявления

Если приоритет одинаковый - применяется последнее правило

``` css
p {
  color: blue;
}

p {
  color: red;
}
```

* будет красный

### 4. !important

``` css
p {
  color: blue !important;
}
```

* перебивает почти всё

* использовать осторожно

## Пример каскада

``` html
<p class="text" id="main">Пример</p>
```

``` css
p { color: blue; }
.text { color: green; }
#main { color: red; }
```

* итог: красный

## Аппаратно-зависимые таблицы стилей

Это стили, которые применяются в зависимости от устройства.

Используются через media queries.

## Media queries

``` css
@media (max-width: 600px) {
  body {
    background: lightblue;
  }
}
```

* стиль применяется на маленьких экранах

### Для телефона

``` css
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
}
```

### Для большого экрана

``` css
@media (min-width: 1200px) {
  .container {
    width: 1000px;
  }
}
```

## 💡 Адаптивный дизайн

Media queries используются для:

- мобильных устройств
- планшетов
- десктопов
